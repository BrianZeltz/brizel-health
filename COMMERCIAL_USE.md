# Commercial Use

## Default Rule

Commercial use rights are **not** included in the repository license.

Brizel Health is published under a noncommercial source-available license for the public repository.

## Examples Of Uses That Need Separate Permission

- paid mobile or desktop apps based on the code
- SaaS or hosted services based on the code
- commercial Home Assistant bundles or appliances shipping this code
- white-label products
- resale or paid redistribution
- integrating substantial parts of the code into a commercial software product

## How To Request Commercial Rights

Please request a separate commercial license before proceeding.

Contact:

- [brian@brizel.de](mailto:brian@brizel.de)

Recommended information to include:

- company or organization name
- intended use case
- whether distribution, hosting, resale, or white-labeling is involved
- expected scale
- requested timeline

## Important Note

Opening a GitHub issue is not a substitute for obtaining commercial permission.

Until a separate commercial agreement exists, the public repository license remains the only granted license.
